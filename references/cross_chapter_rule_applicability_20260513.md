# 跨章节规则适配性问题（Task3）

## 问题本质

规则按"标准章节"设计适用章节，但**报告实际写法不按标准格式**。LLM 在审查某章节时，只能看到该章节的规则，导致假阳性。

## 典型场景

- **C-019 公众参与**：规则适用章节写的是"概述, 公众参与"，但实际报告中公众参与内容在**结论章节**（第十一章）。概述章节只有"环评工作过程"描述，没有明确提及公众参与。
- 结果：概述章节审查时 C-019 规则被传入，但找不到公众参与内容 → 误判为缺陷（假阳性）

## 当前实现分析

```python
    # _chapter_matches_applicable(chapter_name, applicable)
```

### `_chapter_matches_applicable` 匹配规则

- 规则有 `适用章节：概述, 公众参与`
- 审查"结论"章节时：`["概述", "公众参与"]` 与 `"结论"` 做**子串匹配 + 模糊匹配**
- `结论` 不在 `["概述", "公众参与"]` 中 → 返回 `False`
- **C-019 规则根本不会出现在"结论"章节的 prompt 里**

## 三种解决方向

| 方案 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| **A：规则增加适用章节** | 把所有可能包含该内容的章节写入 `适用章节` 字段 | 简单直接，改动小 | 规则描述变复杂 |
| **B：后处理跨章节合并** | 审查完成后，按规则汇总所有章节的发现 | 不改规则和LLM逻辑 | 需额外后处理，判断本身易出错 |
| **C：规则增加兜底描述** | 在规则末尾加"请同时检索其他章节" | 不改结构 | 依赖LLM自觉，效果不稳定 |

## 用户确认方向（2026-05-13）

**方案 A + 升级版 Step2（规则优化建议系统）**：

1. **Step1（A）✅**：把"结论"加到 C-019 各子规则的 `适用章节`（`概述, 公众参与, 结论, 结论与建议`）
2. **Step2（升级）✅**：替代原"规则末尾加兜底描述"，升级为完整的规则优化建议系统：
   - LLM 审查时输出 `rule_coverage`（每规则 has_content 判断）
   - 后处理自动全文检索，生成优化建议推送到 Web 端
   - 管理员采纳后自动更新规则库 + 同步到 skill 目录

**禁止做法**：不要把其他章节的内容塞进某一章节的 prompt（跨章节表格补充已于 2026-05-11 删除）
