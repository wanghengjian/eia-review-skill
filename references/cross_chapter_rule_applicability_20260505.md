# 跨章节规则适配性问题（2026-05-05 讨论）

## 问题现象

C-019（公众参与）规则设计时写适用章节为"概述、公众参与"，
但部分报告将公众参与结论散写在**第十一章（结论与建议）**，
概述章节只写"环评工作过程"不含公众参与判定内容。

**结果**：规则在概述和公众参与两个指定章节都找不到充分内容 → 误判为缺陷（假阳性）。

## 三种解决方案

| 方案 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| **A** | 规则增加"允许章节"字段 | 最干净，规则自述意图 | 需要改代码匹配逻辑 |
| **B** | 后处理阶段跨章节合并同类缺陷 | 不改规则，改聚合逻辑 | 增加后处理复杂度 |
| **C** | 规则末尾增加"兜底检查"描述 | 最简单，LLM 自觉扩展搜索 | 依赖 LLM 自觉，不稳定 |

## 确定方案（2026-05-05）

**方案 A + C 结合**：
- 规则写清楚所有可能包含该内容的章节（如 C-019 注明"结论与建议章节可能含公众参与结论"）
- LLM 层面增加兜底描述，要求跨章节检索

## 典型案例

- **C-019-03 公众参与**：适用章节=概述/公众参与，但结论章节（第十一章）含完整公众参与结论小节
- **C-017 风险物质识别**：第七章为主，但风险物质清单可能在工程分析或结论章节重复出现

## 实现位置

- 规则文本：`reference/审核规则库.md`（各规则的"适用章节"字段）
- LLM prompt：`llm_client.py` 或 `review_chapter` prompt 模板中增加跨章节兜底描述
- **不推荐**在章节 prompt 中塞入其他章节内容（违背"只传同 chapter_num 表格"原则）
