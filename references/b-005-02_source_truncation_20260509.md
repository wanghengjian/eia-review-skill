# B-005-02 根因分析：原始报告书截断 vs 传图截断（2026-05-09）

## 缺陷描述（LLM报告）
> 【B-005-02】表格信息与正文存在矛盾：表1中"本项目情况"列显示实验室废水经（表格内容截断），但正文未对实验室废水处理方案进行说明，且表格中"需配套"后内容缺失，信息不完整。
> 报告摘录：表1 环评类别判定说明（表格0）与正文"3、分析判定相关情况"

## 实际根因
LLM 判断为"传图截断"，但实际根因是**原始报告书本身就截断了**。

## 证据（Table[0] Row3 原始内容）
```
Row3: 生物药品制品制造276 | 全部（含研发中试） | 入驻设有工业废水集中处理设施的产业园区或工业集聚区的下列项目：全部（含研发中试） | 不含单纯药品复配、分装 | 项目涉及生物药品的生产制造，废水经收集处理达标后排放，需配套废水污染防治设 | 本次评价按单项等级最高为"报告书"
```

最后一列原文：`需配套废水污染防治设`（最后两字"设施"丢失）。

同一行文字版正文（段落[41]）写的是：
> 本项目废污水经收集处理达标后排入市政污水管网，最终纳入光明水质净化厂进行后续处理，不直接排入地表水体

**说明**：正文有完整描述，但表格本列确实残缺。LLM 报"正文未对实验室废水处理方案进行说明"是**误判**（正文已说明），但"表格信息不完整"的判断是**准确的**。

## 教训
- LLM 对缺陷根因的判断不可直接采信，需验证
- 报告书摘录中"表格内容截断"的描述有两个可能：**原始残缺** 或 **传图截断**
- 核实方法：直接用 `python-docx` 读 Table cell 的 `.text` 属性，对比 LLM 报告的摘录
- 这类"原文档就残缺"的情况是**真实缺陷**，不是系统bug，无需修复代码
