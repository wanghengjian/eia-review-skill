## 二、三个核心问题

### 问题1：S类细则假阳性（第一章重灾区）

**症状**：R31 第一章新增 7 条缺陷，其中 4 条（S-014、C-004-01、C-005-01、S-028）经核实为假阳性。

| 规则 | 章节 | 实际情况 |
|------|------|---------|
| S-014 | 第一章 | 报告已注明"GB 3095-2012及其2018年修改单"，仍被判缺陷 |
| C-004-01 | 第一章 | 甲醇排放标准DB44/27-2001引用正确 |
| C-005-01 | 第一章 | TVOC Pmax 7.62%判定正确 |
| S-028 | 第一章 | 报告已考虑周边居民区，土壤敏感程度判定正确 |

**根因**：规则描述与LLM实际检查方向不匹配：

| 规则 | 规则本意 | LLM误判方向 |
|------|---------|------------|
| S-014 | 检查"是否进行了大气功能区划判定" | 检查"标准版本是否最新" |
| C-004-01 | 检查"标准号+年代号格式是否正确" | 检查"是否存在更优先的标准" |
| C-005-01 | 检查"评价等级判定依据是否充分" | 检查"标准适用是否正确" |

**优化方案**：在S类规则中增加`⚠️ 判断标准`字段，明确"什么情况不判为缺陷"，以及本规则与其他规则的边界。

### 问题2：第五章缺陷消失（R30有9条→R31有0条）

**症状**：同一份报告书，R30在第五章命中9条规则（经核实大部分是真缺陷），R31在第五章命中0条。

**根因**：`extract_chapters_textutil.py` 的 `_h1_counter` 章节编号分配逻辑在遇到非标准H1时不递增，导致章节编号偏移，规则匹配结果不稳定。

**验证**：在 `review_rules_loader.py` 的 `get_rules_text_for_chapter` 加 debug 日志：
```python
print(f"[DEBUG] chapter_num={chapter_num}, chapter_name={chapter_name}, matched={len(matched)}")
```

**修复**：章节提取时保存 `chapter_mapping.json`，记录每个 chapter_num 对应的原始标题。

### 问题3：14条存疑项无法验证（表格数据提取限制）

**根因**：python-docx 的 `table.rows` 对某些复杂嵌套表格返回空行，无法提取表格数据。

**解法**：用 lxml 直接读 DOCX XML（`scripts/verify_tables.py`）：
```bash
python3 scripts/verify_tables.py report.docx "NMHC,表3.5-14"
python3 scripts/verify_tables.py report.docx --all
```
