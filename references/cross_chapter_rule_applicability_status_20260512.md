# 跨章节规则适配性问题（2026-05-12 讨论未完成）

## 典型案例：C-019 公众参与

**规则 C-019-03 适用章节**：`概述, 公众参与`

**报告实际情况**：
- `概述`章节：只有"环评工作过程"，没有明确写公众参与
- 公众参与内容：实际散在第十一章（结论章节），第十一章有完整的"11.8 公众参与结论"小节

**结果**：规则在概述章节找不到公众参与内容 → 判为缺陷（假阳性）

## 用户提出的三个方案

| 方案 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| A | 规则增加"允许章节"字段，支持多章节 | 最干净 | 规则库维护成本高 |
| B | 后处理阶段做跨章节合并 | 不改规则 | 需要额外处理层 |
| C | 规则末尾增加"兜底检查"描述 | 最省事 | 依赖LLM自觉 |

**用户倾向**：方案 A + 方案 C 结合（规则写清楚所有可能包含的章节 + LLM 自己多翻几个地方找）

## 问题本质

规则按"适用章节"字段匹配到某章节后，只在该章节的文本+表格中查找内容。如果规则实际需要的内容在别的章节，就会产生假阳性。

这不是代码 bug，是**规则描述粒度**与**报告实际写法**之间的语义错配。
