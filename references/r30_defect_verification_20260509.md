## 假阳性分类框架

### Ⅰ类：跨章节假阳性
**特征**：规则要求本章包含某内容，但内容实际在别章节，规则描述不完整。
**处理**：规则加"跨章节兜底"说明（已实施于 C-019）。
**本批次实例**：
- C-019-02（概述公众参与缺失）→ 11.8节已完整说明
- C-019-04（公众意见采纳缺失）→ 11.8节已有

### Ⅱ类：轻微误判（部分遗漏 ≠ 完全遗漏）
**特征**：报告有描述但不完整，LLM夸大了严重程度。
**本批次实例**：
- C-017-01（风险物质识别遗漏）→ 7.1.1提到"废水站存储少量化学品"但未列具体名称，属**部分遗漏**非完全遗漏
- C-017-02（风险潜势分级矛盾）→ 表格式填写P3与正文P4结论不匹配，属**格式错误**非**逻辑错误**
- C-002-05（区域规划环评准入）→ 报告写了"符合"但缺陷说"未提准入要求"，两者语义接近

### Ⅲ类：LLM自创内容
**特征**：LLM在报告中找不到对应内容，凭空编造。
**本批次实例**：无（R26有"硫酸乙醇"案例）

## 核实流程（推荐）

### 步骤1：拉缺陷列表
```bash
sqlite3 ~/.hermes/workspace/eia-review/backend/eia_review.db \
  "SELECT id, rule_id, description, chapter, report_excerpt FROM defects WHERE review_id = 30 ORDER BY chapter, rule_id"
```

### 步骤2：建立章节分布
```python
# 用python-docx提取段落
doc = Document('报告书.docx')
all_text = [''.join(p.itertext()).strip() for p in body.findall('.//{...}p')]
```

### 步骤3：按章节分组 + 关键词验证
按 chapter 分组后，每章节用关键词搜索验证。

**⚠️ 并发限制**：delegate_task 最大 3 个并发子任务，本批次48条分3组（18+16+14）可避免。

### 步骤4：逐条判定框架

```
IF 报告摘录内容在报告中完全不存在:
    → 假阳性（LLM自创）
ELIF 报告摘录在报告其他章节存在:
    → 跨章节假阳性（规则缺跨章节兜底）
ELIF 报告有相关内容但不完整（部分遗漏）:
    → 轻微误判
ELIF 报告相关内容存在但有计算/逻辑错误:
    → 属实
ELIF 报告相关内容完整准确:
    → 假阳性（LLM误判）
```

### 步骤5：严重程度分级
- **极高**：单位/标准号根本性错误，或核心数据矛盾超过10%
- **高**：有明确规范依据的错误
- **中**：规范边缘问题或描述不充分
- **低**：措辞/格式不规范但不影响实质
