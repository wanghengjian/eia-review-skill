# 规则层假阳性消除方法论（R30核实总结）

> 源自：R30 48条缺陷逐条核实（2026-05-09），79%准确率 → 目标88-90%

## 核心发现

R30 的21%误判集中在3类模式：

| 类型 | 数量 | 典型案例 | 根因 |
|------|------|----------|------|
| **跨章节假阳性** | 2条 | C-019-02/04概述章节报错但内容在结论 | 规则适用章节与报告实际章节不一致 |
| **轻微问题升级严重** | 3条 | C-017-01(笼统提及≠完全遗漏)、C-017-02(格式错误≠逻辑矛盾) | 规则无"较轻问题"判断路径 |
| **语义接近被误判** | 1条 | C-002-05(相符性分析 vs 准入要求) | 规则描述模糊，LLM自由裁量过度 |

## 方法论：规则增加"判断标准"段

### 适用场景
当同一规则在多个报告中反复产生**轻微误判**，且误判模式固定时，在规则末尾加 `**⚠️ 判断标准**` 段落。

### 格式模板
```
**⚠️ 判断标准**：
  - 如果[报告有X描述] → 属于"[情况A]"，应判为"[结论A]"
  - 只有当[完全未提及时] → 才判为"[严重结论]"
```

### 实施案例（R30后）

| 规则 | 优化内容 | 预计消除 |
|------|----------|----------|
| B-008-01 | 正文有描述但表格未列 → 较轻，建议核实 | B类1条 |
| C-002-05 | 相符性分析已覆盖准入要求主要方面 | C类1条 |
| C-003-01 | 必须引用生态环境部名录，非深圳市名录 | C类1条 |
| C-017-01 | 笼统提及但未逐项列出 → 较轻，建议补充 | C类1条 |
| C-017-02 | 正文P4表格P3 → 格式不一致，非判定矛盾 | C类1条 |

## 操作流程

1. **识别**：通过缺陷核实发现某规则误判模式固定
2. **定位**：`reference/审核规则库.md` 中对应规则块
3. **加段**：在 `**审核步骤**` 后加 `**⚠️ 判断标准**` 段
4. **验证**：重跑同一报告，对比该规则缺陷数量变化
5. **同步**：push GitHub + 同步到 skill 目录 + 重启后端

## 与其他优化手段的关系

| 手段 | 适用场景 | 优先级 |
|------|----------|--------|
| **规则加判断标准** | 误判模式固定、边界清晰 | P1（首选） |
| **Prompt约束层** | 需要告知LLM概念区分或计算公式 | P2 |
| **后处理过滤** | 规则无法覆盖的特例 | P3（兜底） |
| **规则库拆分** | 审核步骤内部逻辑过于复杂 | P4（慎用） |

## 判断标准撰写原则

1. **先说正常情况，再说例外**：让LLM先匹配正常路径
2. **具体 > 抽象**：`"正文P4表格P3"` 比 `"表格与正文不一致"` 更精准
3. **降级路径明确**：每个标准都给出"升级"和"降级"两条路径
4. **禁止否定式描述**：`不得判为"遗漏"` 改为 `应判为"建议补充"`
